Using Source Code Control
previous|next

You can use SQL Developer to import files and version them. You first need to setup a Subversion repository and create a connection to it. You can see the repository you created under the Versioning Navigator tab. Once you have imported the files into your version control system, you can work on the files. You can browse and edit the files. SQL Developer can track the history of the files and you can compare the recent changes made to the file.
